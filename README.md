# Социальный рейтинг чата

Telegram-бот для одного группового чата. Он молча считает активность участников в течение недели, показывает текущий рейтинг по командам и по воскресеньям публикует итоги.

Бот рассчитан на простой запуск: один Python-процесс, SQLite-файл и long polling. Для маленького чата этого более чем достаточно.

## Что умеет бот

- считает сообщения
- считает полученные реакции
- считает отданные реакции, если Telegram прислал автора реакции
- считает упоминания
- считает пересылки из каналов/пабликов
- считает кружочки
- считает недельную репутацию через `/rep`
- раз в неделю публикует рейтинг и титулы

## Формула очков

```text
score = messages*1 + reactions_received*3 + mentions*2 + reactions_given*0.5 + rep_balance*2
```

Где:
- `rep_balance = rep_plus - rep_minus`

## Команды

- `/top` — рейтинг недели
- `/myrating` — твоя статистика за неделю
- `/summary` — промежуточные итоги недели
- `/rep` — дать участнику `+rep` или `-rep`
- `/about` — что делает бот

## Как работает `/rep`

1. В группе отправляешь `/rep`
2. Бот показывает список известных ему участников этого чата
3. Выбираешь человека
4. Выбираешь `+rep` или `-rep`

Правила:
- нельзя выбрать самого себя
- нельзя выбрать бота
- один человек может держать один активный rep-голос на одного участника в рамках недели
- тот же знак повторно ничего не меняет
- противоположный знак переворачивает голос

Важно:
- бот узнаёт участников, когда видит их активность в чате
- самый надёжный способ “познакомить” бота с человеком — чтобы тот написал хотя бы одно сообщение

## Титулы недели

- `🏆 Король чата` — лучший по общему score
- `🤫 Молчун` — меньше всех сообщений
- `🔥 Любимчик` — больше всех полученных реакций
- `💬 Самый активный` — больше всех сообщений
- `📰 Ньюсмейкер` — больше всех пересылок из пабликов
- `🎥 Блогер` — больше всех кружочков
- `🪫 Беспонтовый` — худший по общему score

## Как бот считает неделю

- используется календарная неделя в часовом поясе `TZ`
- `/summary` показывает текущий срез недели
- официальный weekly-post публикуется по воскресеньям
- титулы сохраняются только при воскресном автопосте

## Переменные окружения

Смотри [.env.example](./.env.example).

Основные:
- `BOT_TOKEN` — токен бота от `@BotFather`
- `TZ` — часовой пояс, по умолчанию обычно `Europe/Moscow`
- `POST_HOUR` — час публикации итогов по местному времени
- `POST_MINUTE` — минута публикации итогов
- `DB_PATH` — путь к SQLite-файлу
- `LOG_LEVEL` — уровень логов

`CHAT_ID` необязателен. Если не задан, бот привязывается к первому групповому чату, где увидит активность.

## Локальный запуск

```bash
python -m venv .venv
source .venv/bin/activate
pip install -e .[dev]
python -m bot.main
```

## Запуск на Railway

1. Залей проект в GitHub
2. Создай новый проект в Railway из репозитория
3. Railway сам соберёт контейнер по `Dockerfile`
4. Добавь volume и смонтируй его в `/data`
5. Добавь переменные:
   - `BOT_TOKEN`
   - `TZ=Europe/Moscow`
   - `POST_HOUR=21`
   - `POST_MINUTE=0`
   - `DB_PATH=/data/bot.sqlite3`
   - `LOG_LEVEL=INFO`

Без volume база SQLite будет теряться после перезапусков и redeploy.

## Настройка Telegram

Перед использованием:

1. Создай бота у `@BotFather`
2. Получи `BOT_TOKEN`
3. Добавь бота в нужную группу
4. Отключи privacy mode у `@BotFather`

Как отключить privacy mode:
- `/setprivacy`
- выбрать бота
- `Disable`

Если privacy mode включён, бот не увидит обычные сообщения и рейтинг будет неполным.

## Что бот считает надёжно, а что с ограничениями

Надёжно:
- сообщения
- кружочки
- пересылки из каналов
- упоминания через text mention
- полученные реакции, если сообщение уже известно боту

С ограничениями Telegram:
- отданные реакции считаются только если Telegram прислал автора реакции
- `@username`-упоминания работают лучше, если бот уже знает этого пользователя
- бот не пересчитывает историю задним числом

## Ограничения текущей версии

- бот работает только с одним активным групповым чатом
- не тянет старую историю сообщений до момента запуска
- не пытается “угадывать” участников, которых ещё не видел
- не пересчитывает статистику после удаления или редактирования сообщений

## Проверка после деплоя

Минимальный smoke test:

1. Напиши пару сообщений в группе
2. Пусть ещё 1-2 участника тоже что-нибудь напишут
3. Проверь `/top`
4. Проверь `/myrating`
5. Проверь `/rep`
6. Проверь `/about`

Если `/rep` пишет, что некого выбирать, значит бот ещё не успел увидеть других участников в этом чате.
